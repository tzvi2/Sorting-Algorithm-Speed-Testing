This site is made for testing the performance of various sorting algorithms in sorting specific arrays. This can be useful for:

  • optimizing a website/app by using the fastest sorting algorithms for the particular data processed

  • calculating the break points (in size) at which various sorting algorithms become faster than counterparts, and perhaps worth using.

Different sorting algorithms can have different runtimes when run with different size inputs. Algorithms with an O(n^2) time complexity can perform faster (in miliseconds) than an algorithm with a "better" big-O time given certain size inputs. This site will allow you to sort arrays with various sorting algorithms in order to find the fastest one for your specific needs. Results are displayed for easy comparison.
